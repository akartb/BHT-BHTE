# BHC + BHTE 安全审计准备文档

**项目**: BHC + BHTE 双层金融生态系统
**版本**: v1.0
**日期**: 2026-03-22
**目标**: 主网上线前专业安全审计

---

## 1. 项目概述

### 1.1 核心架构

| 层级 | 名称 | 描述 |
|------|------|------|
| L1 | BHC | 数字黄金，ProgPoW + ML-DSA |
| L2 | BHTE | 数字现金，zkEVM + 状态通道 |

### 1.2 关键创新

- **ProgPoW 算法**: 抗 ASIC GPU 友好型工作量证明
- **ML-DSA 签名**: NIST 标准化的后量子签名算法 (Level 3)
- **双态 zkEVM**: Kernel Space + User Space 分层执行
- **AuxPoW 合并挖矿**: BHC 矿工同时挖 BHTE
- **状态通道网络**: HTLC + 多跳路由

## 4. 审计重点领域

### 4.1 密码学实现

#### ProgPoW 算法
- 文件: `BHC/src/crypto/progpow.cpp`
- 关注点:
  - DAG 生成逻辑
  - 循环边界条件
  - 内存访问模式

#### ML-DSA 签名
- 文件: `BHC/src/crypto/mldsa.cpp`
- 关注点:
  - 密钥生成随机性
  - 签名验证时序攻击
  - 恒定时间实现

#### 混合签名验证
- 文件: `BHC/src/script/hybrid_sig.cpp`
- 关注点:
  - ECDSA + ML-DSA 验证顺序
  - 失败时的错误处理

### 4.2 共识机制

#### AuxPoW 验证
- 文件: `BHTE/consensus/bhc/auxpow.go`
- 关注点:
  - Coinbase 树验证
  - Merkle 分支验证
  - 链索引计算

#### 合并挖矿协调器
- 文件: `BHTE/consensus/bhc/coordinator.go`
- 关注点:
  - Anti-Sybil 机制有效性
  - 注册频率限制
  - PoW 阈值设置

### 4.3 跨层交互

#### 锚定验证
- 文件: `BHTE/consensus/bhc/anchor.go`
- 关注点:
  - OP_RETURN 格式验证
  - 确认期安全性
  - 状态根提取逻辑

#### SPV 桥接
- 文件: `BHTE/spv/spv_bridge.go`
- 关注点:
  - 区块头同步
  - Merkle 证明验证
  - 重放攻击防护

### 4.4 智能合约

#### L2 结算合约
- 文件: `BHTE/contracts/BHTEsettlement.sol`
- 关注点:
  - 存款逻辑
  - 取款验证
  - 挑战期实现

### 4.5 状态通道

#### HTLC 实现
- 文件: `BHTE/consensus/bhc/channel_network.go`
- 关注点:
  - 哈希锁安全性
  - 时间锁正确性
  - Refund 逻辑

#### 支付通道管理
- 文件: `BHTE/consensus/bhc/state_channel.go`
- 关注点:
  - 余额更新原子性
  - 签名验证
  - 通道关闭安全性

## 5. 审计方法建议

### 5.1 静态分析

- 使用 [Semgrep](https://semgrep.dev/) 扫描代码
- 检查密码学 API 误用
- 检查常见漏洞模式

### 5.2 动态测试

- 模糊测试 (Fuzzing) RPC 接口
- 签名验证边界测试
- 并发安全性测试

### 5.3 形式化验证

- 密码学实现正确性证明
- 状态机转换验证
- 共识协议安全性分析

## 6. 测试建议

### 6.1 单元测试覆盖率目标

| 模块 | 覆盖率目标 |
|------|------------|
| ProgPoW | ≥ 90% |
| ML-DSA | ≥ 95% |
| Hybrid Sig | ≥ 90% |
| AuxPoW | ≥ 85% |
| HTLC | ≥ 90% |

### 6.2 集成测试场景

1. **合并挖矿流程**: BHC 矿工 → AuxPoW → BHTE 区块
2. **跨层锚定**: BHTE 状态根 → BHC OP_RETURN → 验证
3. **状态通道**: 开启 → HTLC 转账 → 关闭
4. **提现流程**: 挑战期 → 欺诈证明 → 逃生舱

### 6.3 压力测试

- 1000+ 并发连接
- 10,000+ 状态通道
- 1M+ 交易吞吐量
