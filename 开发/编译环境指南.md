# BHC C++20 编译环境排烟指南

## 📋 环境要求清单

### 1. 编译器版本要求

| 编译器 | 最低版本 | 推荐版本 | 原因 |
|--------|----------|----------|------|
| GCC | 11.0 | 13.0+ | C++20 完整支持 |
| Clang | 16.0 | 18.0+ | std::span, constexpr |
| MSVC | 19.30 | 19.40+ | Windows 编译 |

### 2. 内存与存储要求

| 资源 | 最低要求 | 推荐配置 | 说明 |
|------|----------|----------|------|
| RAM | 8GB | 16GB+ | 编译时内存消耗大 |
| 交换空间 | 4GB | 8GB | 防止 OOM |
| 磁盘空间 | 20GB | 50GB+ | 包含依赖库 |
| CPU核心 | 4核 | 8核+ | 并行编译加速 |

## 🔧 编译配置选项

### 基础编译命令

```bash
# 创建构建目录
mkdir -p build && cd build

# 配置 (推荐配置)
cmake .. \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_CXX_STANDARD=20 \
    -DCMAKE_CXX_COMPILER=g++-13 \
    -DUSE_PROGPOW=ON \
    -DUSE_MLDSA=ON \
    -DUSE_GUI=OFF \
    -DUSE_TESTS=ON \
    -DUSE_WALLET=ON

# 编译 (根据内存调整并行数)
# 16GB+ 内存: make -j$(nproc)
# 8GB 内存: make -j4
# 4GB 内存: make -j2
make -j$(nproc)
```

### 内存优化编译

```bash
# 低内存环境编译选项
cmake .. \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_CXX_FLAGS="-O2 -g0" \
    -DUSE_GUI=OFF \
    -DUSE_TESTS=OFF

# 单线程编译 (最慢但最稳定)
make -j1
```

## ⚠️ 常见编译问题与解决方案

### 问题 1: C++20 特性不支持

```bash
# 错误信息
error: 'span' is not a member of 'std'

# 解决方案
# 确保使用 GCC 11+ 或 Clang 16+
g++ --version
# 如果版本过低，升级编译器
```

### 问题 2: 内存溢出 (OOM)

```bash
# 错误信息
c++: fatal error: Killed signal terminated program cc1plus

# 解决方案
# 1. 减少并行编译数
make -j2

# 2. 增加交换空间
sudo fallocate -l 8G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 3. 禁用优化 (减少内存占用)
cmake .. -DCMAKE_BUILD_TYPE=Debug -DCMAKE_CXX_FLAGS="-O0"
```

### 问题 3: Boost 库版本不兼容

```bash
# 错误信息
error: 'boost::filesystem' has not been declared

# 解决方案
# 安装 Boost 1.80+
sudo apt-get install libboost1.80-all-dev
# 或从源码编译
wget https://boostorg.jfrog.io/artifactory/main/release/1.84.0/source/boost_1_84_0.tar.gz
tar -xzf boost_1_84_0.tar.gz
cd boost_1_84_0
./bootstrap.sh
./b2 -j$(nproc) install
```

### 问题 4: OpenSSL 版本冲突

```bash
# 错误信息
error: 'EVP_MD_CTX_new' was not declared

# 解决方案
# 安装 OpenSSL 3.0+
sudo apt-get install libssl3 libssl-dev
# 或指定 OpenSSL 路径
cmake .. -DOPENSSL_ROOT_DIR=/usr/local/ssl
```

### 问题 5: ML-DSA 签名大小限制

```cpp
// 在 src/policy/policy.h 中修改
// 原始值
static const unsigned int MAX_STANDARD_P2WSH_STACK_ITEM_SIZE = 80;

// 修改为 (支持 ML-DSA 3293 字节签名)
static const unsigned int MAX_STANDARD_P2WSH_STACK_ITEM_SIZE = 4096;
```

## 🚀 快速启动脚本

```bash
#!/bin/bash
# quick_build.sh - BHC 快速编译脚本

set -e

echo "=== BHC 快速编译脚本 ==="

# 检测系统内存
TOTAL_MEM=$(grep MemTotal /proc/meminfo | awk '{print $2}')
MEM_GB=$((TOTAL_MEM / 1024 / 1024))

# 根据内存设置并行数
if [ $MEM_GB -ge 16 ]; then
    JOBS=$(nproc)
elif [ $MEM_GB -ge 8 ]; then
    JOBS=4
else
    JOBS=2
fi

echo "检测到 ${MEM_GB}GB 内存，使用 ${JOBS} 个并行任务"

# 创建构建目录
mkdir -p build
cd build

# 配置
cmake .. \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_CXX_STANDARD=20 \
    -DUSE_PROGPOW=ON \
    -DUSE_MLDSA=ON \
    -DUSE_GUI=OFF \
    -DUSE_TESTS=ON

# 编译
echo "开始编译..."
time make -j${JOBS}

echo "=== 编译完成 ==="
echo "二进制文件位于: $(pwd)/src/"
```
